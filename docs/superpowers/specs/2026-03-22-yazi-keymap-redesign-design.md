# Yazi Keymap Redesign

日期：2026-03-22

## 背景

当前 `Yazi` 键位已经针对 `Sofle` 左手区做过重排，但高频编辑动作仍然分散在二段前缀中。目标是把最常用的文件操作提升为主层单键，同时保留现有按 `Sofle` 左手区组织的思路。

事实源：
- `nix/home/configs/yazi/keymap.toml`
- `docs/KEYBINDINGS.md`

## 目标

1. 把最常用的编辑动作提升为单键。
2. 保留左手区的工作流分组，不回退到默认 `Yazi`/`Vim` 布局。
3. 尽量保留现有稳定前缀族，降低迁移成本。
4. 把低频功能迁到二段前缀或次高频单键。

## 主层设计

### 保留

- `q`：退出
- `w`：前进到下一个目录
- `e`：返回上一个目录
- `Enter`：打开
- `Space`：选择当前项并下移
- `Up` / `Down` / `Left` / `Right`：移动与目录进退

### 新主层

- `a`：新建文件/目录
- `s`：进入选择模式
- `d`：移到回收站
- `f`：按文件名搜索
- `r`：重命名
- `z`：`fzf` 快速跳转
- `x`：剪切
- `c`：复制
- `v`：粘贴
- `b`：`zoxide` 跳目录
- `.`：显示/隐藏隐藏文件

## 次层设计

### 保持不动

- `g*`：目录跳转
- `t*`：tab 与任务
- `,*`：排序

### 新前缀族

- `l*`：linemode / 显示模式
  - `ls`：size
  - `lp`：permissions
  - `lb`：btime
  - `lm`：mtime
  - `lo`：owner
  - `ln`：none

- `y*`：复制路径
  - `yy`：复制完整路径
  - `yd`：复制目录路径
  - `yf`：复制文件名
  - `ye`：复制无扩展名文件名

- `p*`：低频文件操作
  - `pf`：强制粘贴
  - `pq`：取消当前 yank 状态
  - `p-`：建立绝对路径软链接
  - `p+`：建立相对路径软链接
  - `pz`：永久删除

### 次高频单键

- `o`：交互式选择 opener
- `i`：查看当前文件信息
- `h`：按内容搜索
- `/`：过滤当前列表

## 迁移说明

以下旧位置将被重新分配：

- 旧 `r*` 搜索族迁移到 `f` 主键与 `h` / `/` 补充键
- 旧 `d*` linemode/info 迁移到 `l*` 与 `i`
- 旧 `c*` 路径复制迁移到 `y*`
- 旧 `x*` 低频文件操作拆分到主层与 `p*`
- 旧 `z` / `zz` 跳转改为 `z` / `b`

## 风险

1. 这次是整套 muscle memory 重排，不是小修。
2. `x/c/v/r/d/a` 会变成强主键，旧前缀习惯需要重新适应。
3. 文档和配置必须同步更新，否则会产生明显偏差。
