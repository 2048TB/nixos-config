# Binary Cache Optimization

This repository uses extra binary caches to reduce local build time:

- `https://nix-community.cachix.org`
- `https://nixpkgs-wayland.cachix.org`
- `https://cache.garnix.io`

Current definitions:

- `flake.nix` -> `nixConfig.extra-substituters` and `nixConfig.extra-trusted-public-keys`
- `hosts/outputs/default.nix` -> `binaryCaches` (for runtime/module consumption)

Recommended checks:

```bash
just flake-check
nix flake metadata path:/persistent/nixos-config
```

Troubleshooting:

1. If cache hits are low, confirm network access and key configuration.
2. If builds unexpectedly compile from source, verify substituters and trusted keys are in sync between `flake.nix` and `hosts/outputs/default.nix`.
3. If a cache is temporarily unavailable, retry later or remove that cache entry intentionally in one change set.
